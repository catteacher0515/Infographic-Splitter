# Infographic Splitter MVP 规格说明

## 1. 目标

实现一个本地 MVP 工具，用于把黑白手绘知识类总结图自动拆分成可复用的视觉元素。

第一版只使用 OpenCV 图像处理，不引入 AI、大模型、SAM、OCR 或任何图像语义识别依赖。

目标用户流程：

```text
上传图片
-> 点击开始拆分
-> 自动检测候选元素
-> 预览拆分结果
-> 勾选保留元素
-> 按需重命名文件
-> 导出 ZIP
```

## 2. MVP 输入范围

第一版只支持以下类型的图片：

- 白色或浅色背景。
- 黑色线稿。
- 元素之间有明显留白。
- 知识类总结图。
- 手绘风信息图。

支持上传格式：

- `png`
- `jpg`
- `jpeg`
- `webp`

上传限制：

- 单文件最大 `20MB`。

## 3. 非目标

第一版不支持：

- 复杂插画。
- 照片。
- 遮挡关系。
- 彩色图片的精细拆分。
- PSD 拆层。
- 手动框选。
- 手动合并或拆分元素。
- 透明背景导出。
- AI 自动命名。
- OCR 文字理解。
- Qwen-VL、InternVL、Gemini、GPT Vision 等视觉大模型。
- SAM 或其他分割模型。

这些能力可以作为后续版本预留，但不进入 MVP。

## 4. 独立元素定义

第一版拆分的是“视觉原子”或“视觉组合”，不是完整语义模块，也不是最小笔画。

独立元素定义：视觉上可复用、边界相对清晰、可以单独导出的图像块。

以 `Prompt / Context / Harness / Loop` 示例图为基准，第一版应尽量识别以下元素：

- 顶部序号圆圈：`1`、`2`、`3`、`4` 分别作为独立元素。
- 标题框：`Prompt`、`Context`、`Harness`、`Loop` 分别作为独立元素。
- 等号：每个 `=` 分别作为独立元素。
- 中文说明文字：`一句话`、`一堆资料`、`一套约束`、`自己干活并持续反馈` 分别作为独立元素。
- 横向箭头：每个箭头分别作为独立元素。
- `Prompt` 插图：气泡 + 猫整体作为一个独立元素。
- `Context` 插图：资料堆 + 猫 + 文件夹整体作为一个独立元素。
- `Harness` 插图：小猫 + 被绳子拉的小猫整体作为一个独立元素。
- `Loop` 插图：整个循环图作为一个独立元素。

拆分规则：

- 如果一组相邻线稿共同表达一个场景，第一版按整体元素导出。
- 不把场景拆成过小对象。
- 不把整列内容合并成一个语义模块。

## 5. 检测策略

第一版的检测目标是生成“候选视觉块”，不是理解图像语义。

推荐 OpenCV 处理流程：

```text
原图
-> 灰度化
-> 二值化
-> 反色
-> 形态学合并相邻笔画
-> 轮廓检测或连通域检测
-> 过滤小噪点
-> 可选 padding
-> 排序 bounding boxes
-> 从原图裁剪元素
```

二值图目标：

```text
背景 = 黑
元素线稿 = 白
```

候选 API：

- `cv2.findContours()`
- `cv2.connectedComponentsWithStats()`

实现时可以选择更稳定的方案，但外部行为保持一致：输出候选元素的边界框。

## 6. 可调参数

第一版需要暴露以下参数，便于调试不同图片：

- `min_area`：最小面积过滤，默认 `500px^2`。
- `merge_gap`：相邻笔画合并强度，用于控制文字、线条和场景是否被合成一个元素。
- `padding`：裁剪边距。

说明：

- `merge_gap` 太小会把文字和线稿拆碎。
- `merge_gap` 太大会把相邻元素合并。
- 第一版允许用户调参后重新点击“开始拆分”。

## 7. 排序规则

第一版按视觉阅读顺序排序：

```text
从上到下
同一行从左到右
```

排序后生成默认文件名：

```text
element_001.png
element_002.png
element_003.png
```

第一版不做按列分组排序，也不做语义排序。

## 8. 裁剪规则

每个元素从原图按边界框裁剪：

```python
crop = image[y:y+h, x:x+w]
```

第一版需要支持两种裁剪边距效果：

- 紧凑裁剪：`padding = 0`。
- 留白裁剪：默认 `padding = 10`。

padding 必须限制在原图边界内，不能产生越界裁剪。

第一版导出保留原图背景，不做透明背景。

## 9. 命名规则

第一版不做 AI 自动命名。

默认命名规则：

```text
element_001.png
element_002.png
element_003.png
```

用户可以在预览表格中编辑 `file` 字段。

重命名只需要在导出 ZIP 时生效。

导出时需要处理：

- 自动补齐 `.png` 后缀。
- 清理非法路径字符。
- 避免重名，必要时追加数字后缀。

## 10. manifest.json

ZIP 根目录必须包含 `manifest.json`。

格式：

```json
{
  "elements": [
    {
      "id": 1,
      "file": "element_001.png",
      "x": 120,
      "y": 80,
      "width": 240,
      "height": 180
    }
  ]
}
```

要求：

- 坐标使用原图像素坐标。
- 如果应用了 padding，manifest 记录实际导出的裁剪框。
- manifest 只包含用户勾选导出的元素。

## 11. ZIP 导出结构

导出文件为 `export.zip` 或带 session 标识的等价文件名。

结构：

```text
export.zip
|-- assets
|   |-- element_001.png
|   |-- element_002.png
|   `-- element_003.png
`-- manifest.json
```

只导出用户勾选保留的元素。

## 12. Gradio 页面要求

第一版使用 Gradio 实现本地页面。

必须支持：

- 上传单张图片。
- 点击按钮开始检测和拆分。
- 显示原图预览。
- 显示候选元素预览。
- 显示元素表格。
- 勾选或取消勾选导出元素。
- 编辑导出文件名。
- 导出 ZIP。

推荐表格字段：

- `selected`
- `file`
- `x`
- `y`
- `width`
- `height`

检测失败时第一版只支持：

- 调整参数后重新检测。
- 取消勾选误检元素。
- 修改文件名。

第一版不支持手动框选补漏。

## 13. 推荐目录结构

```text
project/
|-- app.py
|-- detector.py
|-- splitter.py
|-- exporter.py
|-- assets/
`-- output/
```

模块职责：

- `app.py`：Gradio 页面和流程编排。
- `detector.py`：OpenCV 预处理、候选检测、过滤和排序。
- `splitter.py`：裁剪、默认命名和元素元数据生成。
- `exporter.py`：ZIP 打包、文件名清理和 manifest 生成。
- `assets/`：可选示例图或静态资源。
- `output/`：临时输出、裁剪图片和 ZIP 文件。

## 14. 依赖范围

第一版仅依赖：

```text
opencv-python
pillow
gradio
```

允许使用 Python 标准库：

- `json`
- `zipfile`
- `pathlib`
- `tempfile`
- `uuid`
- `shutil`

## 15. MVP 验收标准

MVP 完成时必须满足：

- 可以通过 Gradio 上传支持格式的图片。
- 可以点击按钮自动检测候选元素。
- 可以预览候选元素。
- 候选元素按“从上到下、同一行从左到右”排序。
- 默认文件名按 `element_001.png` 递增。
- 可以取消勾选元素。
- 可以编辑导出文件名。
- 可以导出 ZIP。
- ZIP 包含 `assets/` 和 `manifest.json`。
- manifest 只包含已导出的元素。
- 裁剪图保留原图背景。
- 不引入 AI、大模型、SAM、OCR 或其他语义识别依赖。

参考图验收目标：

- 能检测出 `Prompt / Context / Harness / Loop` 示例图中的主要视觉原子和视觉组合。
- 输出元素数量大致在 20 个左右。
- `Harness` 的两只猫 + 绳子场景作为一个元素。
- `Loop` 的循环图作为一个元素。
- 中文说明文字作为独立元素导出。

第一版允许少量误检或漏检，只要用户能通过取消勾选避免错误导出。

## 16. 后续阶段预留

### V2: 更好的自动命名

示例：

```text
speech_bubble.png
folder.png
loop_arrow.png
```

可能需要 OCR 或视觉模型，但 MVP 不接入。

### V3: 智能合并和拆分

示例：

```text
Prompt 标签 + 聊天框 + 猫 -> Prompt 模块
```

可能包括手动框选、合并、拆分和几何规则优化。

### V4: AI 识别

可能接入：

```text
Qwen-VL
InternVL
Gemini
GPT Vision
```

用于语义理解和内容感知命名。

### V5: 透明背景导出

透明背景导出需要前景 mask，并处理纸张纹理、抗锯齿和边缘清理。

此能力明确不进入 MVP。
